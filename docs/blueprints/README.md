# QuantIAN Architecture Blueprints

Following the blueprinting notation from the course's Architecture Blueprinting
module, each view below includes a **Legend Box** identifying the architecture
domain, scope, level of abstraction, state, authorship, and blueprint type.

Views in this folder (in descending order of abstraction):

| # | View | File | Purpose |
|---|------|------|---------|
| 1 | Presentation | [01-presentation.md](01-presentation.md) | One-diagram executive summary |
| 2 | Conceptual   | [02-conceptual.md](02-conceptual.md)   | Bounded components and relationships |
| 3 | Logical      | [03-logical.md](03-logical.md)         | Services, APIs, schemas, data flow |
| 4 | Physical     | [04-physical.md](04-physical.md)       | Cloud-specific deployment resources |

The draw.io sources in [../diagrams/](../diagrams/) are the original working
files. The views in this folder are authored in **Mermaid**, which GitHub
renders directly on the page, so diagrams stay versioned, diff-able, and
embed-able without requiring a graphics editor.
